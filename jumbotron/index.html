By having this, we can manage the users which we are created and add them into groups and assign them roles in their groups or projects. By this, we can manage them by giving them permissions on their access to AWS resources. By this, every user has their unique individual access to their AWS services and their required resource. With IAM Multifactor Authentication enabled, a user trying to access an AWS resource will be prompted for normal authentication (user name and password), but also an authentication code available only through their MFA-configured device.

It allows you to provision compute resources, like EC2 instances and RDS deployments, inside isolated virtual networks. By having this you will get complete control over network traffic (inbound and outbound). By using VPC’s we can restrict access to or from the internet. By using Virtual Private Network (VPN) connections, you can connect on-premise servers directly to your cloud-based VPC, bypassing public networks.

We can create firewall rules controlling incoming and outgoing traffic at the instance level. It can restrict traffic by protocol type (TCP, UDP, ICMP), IP address, and port.
Access Control Lists (ACLs) work at the network subnet level. Network ACLs can be especially useful in the prevention of DDOS attacks when you have a particular need to blacklist traffic from specific IP addresses.

AWS provides Data encryption for EBS volumes, S3 buckets, and Relational Database Service (RDS) and Glacier data stores.
When you create an encrypted EBS volume and attach it to an instance, data on the volume, disk I/O, and snapshots created from the volume, are all encrypted.
When so configured, AWS encrypts each S3 object with a unique key. Amazon S3 server-side encryption uses one of the strongest block ciphers available – 256-bit Advanced Encryption Standard (AES-256).
RDS generates an SSL certificate for each DB Instance. Once an encrypted connection is established, data transferred between the DB Instance and your application will be encrypted during transfer.

Direct Connect provides a private and secure high-bandwidth network connection.
You can use AWS Direct Connect to establish a private virtual interface between your on-premise network and your Amazon Virtual Private Cloud.

It provides you with a history of all API calls made against your account resources, including API calls made via the AWS Management Console, SDKs, and command-line tools.

AWS Trusted Advisor inspects your AWS environment and makes recommendations for saving money, improving system performance and reliability, or closing security gaps.
Trusted Advisor can provide a highly effective summary of your overall Amazon Web Services security profile.
